DVE.prototype.arc()

문법

let ret = object.arc(rx, ry, xar, laf, swf, x, y[, rel]);

파라미터

이름 형식 설명
rx, ry <length>

타원의 호 모양을 결정짓는 x 축 반지름, y 축 반지름의 크기입니다.

그려질 호는 반지름 rx, ry의 크기를 갖는 타원의 일부분입니다.
xar <degree>

rx, ry의 타원을 원점(0, 0)에 놓여 있다고 할 때, x 축에서 양의 방향으로 주어진 각의 크기만큼 타원을 회전시킵니다.

타원이 회전하면 그려질 호도 회전을 하게 됩니다.
laf {0|1}

large-arc-flag를 의미하며, 0이면 작은 호를 1이면 큰 호를 그립니다.
swf {0|1}

sweep-flag는 1인 경우 시작점에서 끝점으로 호를 이루는 점들 중에서 양의 각 방향의 점을 그립니다. 0인 경우는 음의 각 방향의 점들을 그리게됩니다.
x, y <coordinate>

그려질 호의 끝점을 나타냅니다.
rel Boolean

옵션. rel(relative) 값이 true이면, 주어진 위치는 현재 위치의 상대 위치로 인식합니다.

반환

DVE 객체를 반환합니다.

설명

'arc' 메소드는 SVG path 요소의 'elliptical arc' 커맨드를 나타냅니다.

Elliptical arc는 주어진 두 끝점을 거치는 호를 그립니다.

XML로 호를 그릴 때는 A명령어를 사용합니다.

현재 위치를 시작점으로하고 A(a는 상대) 명령어의 마지막 점의 위치를 끝점으로 인식합니다.

명령어는 다음의 파라미터를 갖습니다.

(rx,ry x-axis-rotation large-arc-flag sweep-flag x,y)

rx,ry x-axis-rotation은 호가 그려질 원의 모양을 결정합니다.

large-arc-flag는 큰 호를 그릴지 작은 호를 그릴지 결정합니다.

sweep-flag는 1인 경우 시작점에서 끝점으로 호를 이루는 점들 중에서 양의 각 방향의 점을 그립니다. 0인 경우는 음의 각 방향의 점들을 그리게됩니다. 일반적으로 양의 각이란 시계 반대 방향으로 측정하는 각의 크기를 말하며, SVG 좌표계에서는 y 축이 아래를 향하기 때문에 일반 평면 좌표계와 달리 위, 아래가 뒤집힌 반대가 됩니다. 그러므로 양의 각은 시계 방향으로, 음의 각은 시계 반대 방향으로 이해하면 됩니다.
