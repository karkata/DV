DVE.prototype.fsize()

문법

var ret = dve.fsize(size)

파라미터

이름 형식 설명
size <absolute-size>|<relative-size>

폰트의 크기입니다.

반환

DVE 객체를 반환합니다.

설명

fsize 메소드는 렌더링이 되는 text, tspan 요소에서 텍스트 크기를 정합니다.

<svg width="400" height="200">
    <text x="20" y="50" font-size="0.65em">This property refers to thesize fo the font from baseline.</text>
    <text x="20" y="100" font-size="2em">This property referes to the size of the font from baseline.</text>
</svg>

var svg = DV.svg().size(400,200).into("dv-01");
DV.text().txt("This property refers to thesize fo the font from baseline.").xys(20,50).fsize("0.65em").into(svg);
DV.text().txt("This property refers to thesize fo the font from baseline.").xys(20,100).fsize("2em").into(svg);

결과적으로 두 문장의 폰트 크기 차이가 없음을 알 수 있습니다. 그러나 폰트의 크기가 차이가 없는 이유는 이 페이지의 CSS에서 * 지시자로 13px로 잡혀 있기 때문인데, font-size 속성이 인라인 속성이므로 우선 순위가 높을 줄 알았으나, 크롬에서는 그렇지 않은 것 같습니다. 이런 경우에는 css 메소드를 사용하여 인라인 CSS를 사용합니다.

var svg = DV.svg().size(400,200).into("dv-02");
DV.text().txt("This property refers to thesize fo the font from baseline.").xys(20,50).css("font-size","0.65em").into(svg);
DV.text().txt("This property refers to thesize fo the font from baseline.").xys(20,100).css("font-size","2em").into(svg);
